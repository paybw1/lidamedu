## 0. 한 눈에 보는 정리

PPT 가 그리는 학습 플랫폼의 핵심을 5.4 관점으로 좁히면 다음 한 줄로 요약된다.

> **"조문·판례·문제 3 entity 를, 체계도/조문/연도 라는 동일한 정렬축으로, 같은 우측 보조 패널(즐겨찾기·메모·하이라이트·질의응답·관련자료·코멘트) 과 함께 어디서 들어오든 끊김 없이 잇는다."**

### 5.4 가 반드시 지켜야 할 5가지 원칙

| # | 원칙 | 지키지 못하면 |
|---|------|-------------|
| 1 | **3 entity 대칭성** — 조문·판례·문제는 우측 패널·즐겨찾기·메모·하이라이트가 같은 컴포넌트로 작동 | 사용자가 entity 마다 새 인터페이스 학습 → 학습 흐름 단절 |
| 2 | **유기적 양방향 링크** — 조문 ↔ 판례, 조문 ↔ 문제 지문, 판례 ↔ 문제, 문제 지문 ↔ 조문/판례 | 어디서 진입하든 곧장 이동한다는 차별점이 무너짐 |
| 3 | **체계도 vs 조문 순서 토글** — 트리 정렬을 사용자가 즉시 전환 | 학습자별 멘탈 모델(논리 vs 조문 순) 강제 |
| 4 | **분류 다층 라벨** — 기출/변형/예상, 단답/박스/사례, 긍정/부정, 단원/종합 등 | 검색·필터·통계가 거짓이 됨 |
| 5 | **정답률 기반 난이도** — 사용자 정답률(전체+개인) 으로 난이도가 동적 계산 | 콘텐츠 진단·약점 분석 불가능 |

## 1. 도메인 모델 (이 메뉴가 다루는 데이터)

> 본 절은 5.4 화면이 의존하는 데이터의 모양을 정리한다. 정확한 컬럼 정의는 `docs/db-schema.md` 에 옮긴다 (해당 문서 작성 task 분리).

### 1.1 Entity 3종 + 부속

```
법령 LAW
  └ 조문 ARTICLE   ← 조/항/호/목 4단계 트리 (PPT slide 11·15)
       ├ article_revision   (개정 스냅샷, 시점 조회)
       ├ 관련조문 링크       (article ↔ article)
       ├ 정오문제 연동      (article ↔ problem 지문)
       ├ 코멘트(평석)
       ├ 강의노트/영상 첨부
       └ Q&A 스레드

판례 CASE   ← (Slide 21·23)
  ├ 분류 (특허/디자인/상표) + 체계도 + 조문
  ├ 일반 / 전원합의체
  ├ 법원, 선고일, 사건번호, 사건명
  ├ 판결요지 (제목/내용)
  ├ 판시이유 + 판결전문 PDF
  ├ 코멘트(출처/내용)
  ├ 관련문제: 1차 기출, 2차 기출, 2차 예상
  ├ 관련논문 / 관련기사 (PDF/링크)
  ├ 강의노트/영상
  └ Q&A 스레드

문제 PROBLEM   ← (Slide 17·19·27·29)
  ├ 시험 구분: 1차 객관식 / 2차 주관식
  ├ 출제 구분: 기출 / 기출변형 / 예상 / 모의고사
  ├ 1차 객관식 분류: 단원/종합 · 단답형/박스형/사례형 · 긍정형/부정형
  ├ 2차 주관식 분류: 키워드, 사례·논점
  ├ 정답률 → 난이도 (상 0–40 / 중 41–80 / 하 81–100)
  ├ 1차: 지문별 색인 + 정오문제 연동
  ├ 1차: 해설(지문별 O/X) + 분류(조문/판례/실무·이론) + 링크
  ├ 2차: 채점기준 + 모범답안 + 채점결과(소문제·전체)
  ├ 강의노트/영상 (동영상 풀이)
  └ Q&A 스레드
```

### 1.2 사용자 학습 데이터 (모든 entity 에 polymorphic)

- `user_bookmarks` — ♡ 1~5단계 (PPT 의 `♡♡♡♡♡` 5하트 표기), 클릭 시 **메모 자동 첨부 가능**
- `user_memos` — 자유 메모 (즐겨찾기 클릭 시점 + 별도)
- `user_highlights` — 본문/요지/판시/지문/해설 등 텍스트 영역 어디서나
- `user_qna_threads` — 질문·답변 + 답변자 평가 (상/중/하)
- `user_problem_attempts` — 객관식 시도(정답 여부, 회독수), 주관식 답안 업로드 + 채점

### 1.3 분류·정렬 axis (모든 entity 가 공유)

| Axis | 조문 | 판례 | 문제 |
|---|---|---|---|
| **체계도 순서** | 발명의 성립성 → 특허요건 → … | 같은 체계도 노드에 매핑 | 같음 |
| **조문 순서** | 제 1장 총칙 → 제 2장 특허요건 → … 제 12장 벌칙 | 관련 조문에 매핑 | 출제 조문에 매핑 |
| **연도 순서** | (해당 없음) | 선고일 | 출제일/회차 |
| **세분류** | `특허요건 > 신규성` 등 leaf | 같은 leaf 매핑 | `제 29조 1항` 등 leaf |

→ **"체계도/조문 순서 토글"은 5.4.A 의 글로벌 상태**. 조문·판례·문제 탭 모두에 적용된다.

## 2. 공통 시스템 (5.4 전반에 깔리는 횡단 컴포넌트)

이 절의 내용은 5.4.A.1 / 5.4.A.2 / 5.4.A.3 에 **모두 동일하게 작동**해야 한다. 공통 컴포넌트로 한 번 구현해 entity type 만 prop 으로 받도록 한다.

### 2.1 우측 보조 패널 (Right Drawer / Right Panel) — 핵심 UX

PPT 의 모든 entity 상세 화면은 **본문 우측에 탭 패널**을 둔다. 탭 구성은 entity 별로 다르지만 공통 베이스가 있다.

| 탭 | 조문 | 판례 | 1차 객관식 | 2차 주관식 |
|---|:---:|:---:|:---:|:---:|
| 정오문제 | ✅ | — | — | — |
| 관련문제 | — | ✅ | — | — |
| 관련자료 (강의노트·영상) | ✅ | ✅ | ✅ | ✅ |
| 관련논문·기사 | — | ✅ | — | — |
| 코멘트(평석) | ✅ | ✅ | (해설로 대체) | — |
| 채점기준·모범답안·결과 | — | — | — | ✅ |
| 즐겨찾기 ♡ | ✅ | ✅ | ✅ | ✅ |
| 메모 | ✅ | ✅ | ✅ | ✅ |
| 질의응답 (Q&A) | ✅ | ✅ | ✅ | ✅ |

→ 우측 패널 **shell** 컴포넌트를 한 번 구현 (`<RightPanel target={{type, id}} tabs={[...]}/>`).

### 2.2 즐겨찾기 ♡ 5단계 + 메모

PPT slide 11/16/20/24/30 의 `♡♡♡♡♡` 표기는 5단계 평점 즐겨찾기 의미.

- 토글이 아니라 **0~5 stars** 형태의 평점
- ♡ 클릭 시 **메모 입력창**이 즉시 열림 (선택적 메모)
- 메모는 ♡ 단계 변경 이력에 묶여 timeline 형태로 누적

### 2.3 하이라이트

- 조문 본문, 항/호/목, 판결요지, 판시이유, 코멘트, 문제 지문, 해설, 모범답안 — 모든 텍스트 영역에서 마우스 드래그 → 색 선택 → 저장
- 색상은 4종 권장 (초록·노랑·빨강·파랑) 의미 사용자 자유
- 콘텐츠가 개정/수정되면 `start_offset / end_offset + content_hash` 비교 → 범위 유효성 자동 판정 (CLAUDE.md Non-negotiable #9 반영)

### 2.4 질의응답 (Q&A)

PPT slide 16/20/24/30 의 동일 패턴이 4번 반복됨 → **하나의 컴포넌트로 통합**.

흐름:

1. 검색 (기존 질문/답변 검색)
2. 새 질문 작성 → 답변자(강사·운영자)에게 **이메일 또는 카카오** 알림
3. 답변자 링크 클릭 → 답변 작성 → 질문자에게 알림
4. 답변자가 **질문 수준 평가** (상 / 중 / 하)
   - 상: 기본 지식 있고 요구사항 정확히 파악
   - 중: 기본 지식 있으나 요구사항 파악 못함
   - 하: 기본 이론 부족
5. 평가는 **사용자 진단/분석 (Slide 44)** 에 사용 (질의수준 지표)

### 2.5 강의노트 · 동영상 첨부

- 모든 entity 에 강의노트 파일(PDF/MD) + 강의 동영상 링크 첨부 가능
- 사용자 화면에서 **파일 뷰어 임베드** + **외부 영상 플레이어** (YouTube/Vimeo/자체 호스팅)
- 첨부는 강사·운영자 권한 (5.7 Admin 영역)

### 2.6 코멘트 (평석 / 학습 자료)

- 강사·운영자가 작성하는 학습 자료
- 조문: 해당 조문에 대한 학습 코멘트
- 판례: **출처 표시 + 내용** 구조 (출처와 내용을 별도 필드로 구분 — Slide 24)
- 하이라이트 가능

### 2.7 업데이트 정보 / 시점 조회

- 모든 entity 는 개정/수정 이력 보존 (조문 `article_revisions`, 판례·문제는 일반 `updated_at` + 큰 변경 시 revision)
- "최신 정보" 메뉴(5.3) 와 같은 데이터를 사용 — 5.4 진입 시 **본문 상단에 최근 업데이트 배지** 노출
- 조문은 `?at=YYYY-MM-DD` 시점 조회 (이미 `feat-4-A-108` 등록)

### 2.8 사용자 진단 / 분석 (Slide 44)

5.4 가 만들어내는 학습 데이터:

| 지표 | 산출 방식 | 노출 위치 |
|---|---|---|
| 정오문제 정답률 | 지문별 O/X 정답 / 시도 수, 유형(조문·판례·이론) 분리 | 조문 우측 패널, 대시보드 |
| 객관식 정답률 | 문제별 정답 / 시도 수, 유형(단답/박스/사례) 분리 | 문제 풀이 결과, 대시보드 |
| 모의고사 정답률 | 회차별 점수 + 평균 비교 | 모의고사 결과 화면 |
| 질의응답 수준 | 답변자 평가(상/중/하) 평균 | 학생 프로필 / 강사 모니터링 |
| 회독수 | 같은 entity 진입 횟수 | 사용자 프로필 |
| 개인 vs 전체 정답률 | 백분위 + 차이 | 결과/대시보드 |

산출은 `feat-000-014` 학습 진도 미들웨어 + `feat-000-015` 일별 집계에서 처리.

## 3. 5.4.A — 법률 과목 학습 허브 (공통)

### 3.1 라우트 & 화면 구조

```
/subjects/:subject              ← 과목 허브 (탭: 조문/판례/문제)
/subjects/:subject?tab=articles
/subjects/:subject?tab=cases
/subjects/:subject?tab=problems

/subjects/:subject/articles/:articlePath    ← 조문 뷰어 (3분할)
/subjects/:subject/cases/:caseId            ← 판례 상세 (3분할)
/subjects/:subject/problems/:problemId      ← 문제 상세 (1차/2차 분기)
/subjects/:subject/quiz/runner?config=...   ← 객관식 풀이 Runner
/subjects/:subject/essay/runner?config=...  ← 주관식 답안 작성 Runner
```

`subject` 슬러그: `civil` / `patent` / `trademark` / `design` / `civil-procedure`

### 3.2 과목 허브 레이아웃 (`feat-4-A-001`)

```
┌─────────────────────────────────────────────────────────────┐
│ [과목명: 특허법]    [3탭: 조문 │ 판례 │ 문제]    [헤더]      │
│ • 진도: 72%                                                 │
│ • 최근 개정 배지 (있다면)                                   │
│ • 체계도 / 조문 순서 토글  (글로벌 상태)                     │
├─────────────────────────────────────────────────────────────┤
│ (탭 컨텐츠)                                                 │
│   - 조문: 트리 + 본문 + 우측 패널                            │
│   - 판례: 색인 → 상세                                       │
│   - 문제: 색인 → 상세 → 풀이 Runner                          │
└─────────────────────────────────────────────────────────────┘
```

- 탭 상태는 URL 쿼리 (`?tab=`) 동기화 (`feat-4-A-003`)
- 5 법률 과목이 동일 셸 공유

### 3.3 글로벌 상태 — 정렬축 토글 (신규: `feat-4-A-004`)

## 4. 5.4.A.1 — 조문 탭

### 4.1 데이터 모델 보강 (Slide 11·15·16 반영)

| 필드 | 의미 | PPT 출처 |
|---|---|---|
| `article.systematic_path` | 체계도 분류 (e.g. `특허요건 > 특허를 받을 수 있는 발명 > 진보성`) | Slide 19 |
| `article.statutory_path` | 조문 분류 (e.g. `제 2장 > 제 29조 > 제 29조제 2항`) | Slide 19 |
| `article.importance` | 중요도 ★ 1~3 | Slide 15 |
| `article.title_only_collapsible` | 제목만 보이게 가능 여부 | Slide 16 |
| `article_links[]` | 관련조문 링크 (양방향) | Slide 15 |
| `article_revision.kind` | 본문/해설/주석 등 종류 | Slide 11 |

### 4.2 조문 트리 (`feat-4-A-103` 보강)

- **두 모드**: 체계도 / 조문 순서 (글로벌 토글에 반응)
- 트리 노드: 편 → 장 → 절 → 조 → (필요 시 항/호/목)
- 노드별 표시: 진도 마커(읽음/미열람), 즐겨찾기 ♡ 합계 별, 중요도 ★, 최근 개정 점
- "최근 학습"·"미열람 권장" 탑 카드 (`feat-4-A-104`)
- 트리 검색 (`feat-4-A-109`)
- 큰 법(민법 1118조) lazy-load (`feat-4-A-110`)

### 4.3 조문 식별자 (`feat-4-A-102`)

- 양방향 변환 헬퍼: `특허법 제 29조 제 1항 제 2호 가목` ↔ `{law:'patent', article:29, clause:1, item:2, sub_item:'가'}` ↔ URL slug
- 조문 본문 표시 시 `法 81의 3③` 같은 약식 표기도 파싱·렌더링 (Slide 15 의 관련조문 링크)

### 4.4 조문 뷰어 — 3분할 (`feat-4-A-105`)

